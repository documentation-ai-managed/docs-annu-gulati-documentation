## Version Management

Keep track of changes and maintain multiple versions of your documentation simultaneously.

<Steps>
  <Step title="Create Version" icon="plus">
    Start a new version branch for major updates or releases.
  </Step>
  <Step title="Edit Content" icon="edit">
    Make changes to your documentation in the version branch.
  </Step>
  <Step title="Review & Merge" icon="check-circle">
    Review changes and merge them into the main branch.
  </Step>
</Steps>

<ExpandableGroup>
  <Expandable title="What are versions for?" default-open="false">
    Versions allow you to maintain documentation for different product releases or maintain drafts separate from published content.
  </Expandable>
  <Expandable title="How do I switch versions?" default-open="false">
    Use the version selector in the project dashboard to switch between different documentation versions.
  </Expandable>
</ExpandableGroup>

<CodeGroup tabs="Git,Platform UI">
  ```bash
  # Using Git integration
  git checkout -b v2.0-docs
  # Make changes
  git commit -m "Update API docs for v2.0"
  git push origin v2.0-docs
  ```
  ```bash
  # Platform version management
  annu-docs version create v2.0
  annu-docs version switch v2.0
  annu-docs publish
  ```
</CodeGroup>

## Collaboration Features

Work together seamlessly with built-in collaboration tools designed for technical teams.

<Steps>
  <Step title="Invite Collaborators" icon="users">
    Add team members to your project with appropriate permissions.
  </Step>
  <Step title="Real-Time Editing" icon="zap">
    Edit documents simultaneously with conflict resolution.
  </Step>
  <Step title="Review Process" icon="eye">
    Submit changes for review before publishing.
  </Step>
</Steps>

<Expandable title="Permission Levels" default-open="false">
  - **Owner**: Full access to project settings and billing
  - **Editor**: Can create and edit content
  - **Reviewer**: Can review and approve changes
  - **Viewer**: Read-only access to published content
</Expandable>
